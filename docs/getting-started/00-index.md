# Getting Started

Neops is built from several components that work together — the workflow engine, function-block workers, the remote lab, the web client, and supporting services.
Where you start depends on your role.
If you are new to the platform, read **[Concepts → How Neops operates](../concepts/20-how-neops-operates.md)** first to build a system-level mental model, then pick the entry point that matches what you need to do.
