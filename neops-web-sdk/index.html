    neops-web-sdk

    Angular monorepo producing 7 publishable @zebbra/ NPM library packages for the neops network automation platform. Provides the shared foundation for all neops web frontends – UI components, domain services, form renderers, and build artifacts.

    Quick Start

    Prerequisites

    • Node.js 20.19 (see .nvmrc)
    • GitHub Packages authentication for @zebbra scope

    Setup

    # Authenticate with GitHub Packages NPM registry
npm login --scope=@zebbra --registry=https://npm.pkg.github.com

# Install dependencies (also builds schematics via postinstall)
npm install

# Start demo app (http://localhost:4201)
npm start

    Verify your setup: npm run lint && npm test && make scan-bad-imports

    The demo app expects a neops-core backend running at http://localhost:8000/graphql.

    Available Commands

    Command Description
    npm start Demo app on port 4201
    npm run start:prod Demo app with production config
    npm run build Build the demo application
    npm test Unit tests (Karma/Jasmine)
    npm run test:ci Headless Chrome tests with coverage
    npm run lint ESLint (8GB memory limit)
    npm run lint:fix Auto-fix lint issues
    npm run format Check Prettier formatting
    npm run format:fix Fix Prettier formatting
    make libraries Build all 7 libraries in dependency order
    make publish Publish all libraries to GitHub Packages
    make dry-publish Test publish (dry run)
    make scan-bad-imports Check for incorrect cross-library imports
    make typedoc Generate API documentation
    npm run generate:icons Generate Carbon icon loaders
    ng g neops-schematics <name> Scaffold a new dashboard card component

    Architecture

    Monorepo Layout

    projects/zebbra/
  ngx-neops-icons/            # Icon renderer (Carbon icons)
  ngx-jsonforms-carbon/       # JSON Forms renderers (Carbon-styled)
  ngx-neops-app-services/     # Cross-cutting services
  ngx-neops-client/           # Domain services + GraphQL client
  ngx-neops-app-components/   # UI component library (largest library)
  ngx-neops-sdk-artifacts/    # Build config, webpack, nginx, styles
  ngx-neops-storage-client/   # File storage client
src/                          # Demo application
neops-schematics/             # Angular schematics for scaffolding

    Package Inventory

    Libraries must be built in dependency order (enforced by make libraries):

    # Package NPM Name Description
    1 ngx-neops-icons @zebbra/ngx-neops-icons Icon renderer wrapping Carbon icons with lazy loading, fallback, and tooltip support
    2 ngx-jsonforms-carbon @zebbra/ngx-jsonforms-carbon JSON Forms renderers styled with Carbon Design System (text, number, date, autocomplete, array, categorization, etc.)
    3 ngx-neops-app-services @zebbra/ngx-neops-app-services Alert, dialog, notification, download, health check, auto-refresh, and connectivity services
    4 ngx-neops-client @zebbra/ngx-neops-client Auth, GraphQL module, config, context, and entity services (Device, Interface, Client, Group, Task, Execution, Location, Scope, Topology, Report)
    5 ngx-neops-app-components @zebbra/ngx-neops-app-components Generic tables, dashboard cards (many types), header, task cockpit, network diagrams, Monaco editor, map components
    6 ngx-neops-sdk-artifacts @zebbra/ngx-neops-sdk-artifacts Custom webpack config, nginx SPA routing, Docker entrypoint, IBM Plex fonts, SCSS base styles
    7 ngx-neops-storage-client @zebbra/ngx-neops-storage-client File upload/download/tagging client for neops-storage, with file browser UI

    Key Patterns

    ElementSearch: All entity services implement this interface for paginated, sorted, filterable GraphQL queries. Tables consume ElementSearch instances to render data consistently.

    DynamicTableConfig: JSON-serializable configuration that generates entire data tables from JSON files. DynamicTableConfigService reads configs and builds NeopsTableConfiguration + DynamicTableModel pairs.

    Dashboard Cards: Registered via DashboardModule.withCards() and rendered in a gridster grid. Card types include markdown, query, status, charts, maps, network diagrams, and task selectors.

    State Management: No NgRx. State flows through RxJS BehaviorSubject patterns. ContextService holds app-wide state (scope, query filters, task context). Auth managed via AuthService.

    Auth: Dual mode – GraphQL username/password login and OIDC (Keycloak). JWT tokens stored in localStorage with Bearer prefix. Token refresh runs every 2 minutes.

    Configuration

    Runtime Environment Variables

    Variable Default Purpose
    FRONTEND_GRAPHQL_ENDPOINT /graphql GraphQL API endpoint
    FRONTEND_FILEAPI_ENDPOINT / File storage API endpoint
    NEOPS_VERSION develop Build version (set at Docker build time)
    NEOPS_WEB_ENVIRONMENT prod Controls which build output to serve

    Docker

    # Production build
export NPM_TOKEN=ghp_XXX
docker build --secret id=npm_token,env=NPM_TOKEN -t neops-web-sdk .

# Run CI (lint + test + library build)
docker build --target run-ci --secret id=npm_token,env=NPM_TOKEN -t neops-web-sdk:ci .

# Run container
docker run -p 8080:8080 neops-web-sdk

# With custom backend
docker run -p 8080:8080 -e FRONTEND_GRAPHQL_ENDPOINT=http://localhost:8000/graphql neops-web-sdk

    Publishing

    Libraries are published to GitHub Packages NPM registry (https://npm.pkg.github.com) under the @zebbra scope. Version placeholders (0.0.0-placeholder) are replaced by CI during the build pipeline.

    make libraries      # Build all 7 libraries
make dry-publish    # Verify publish (dry run)
make publish        # Publish to GitHub Packages

    Contributing

    Default branch: develop. Branch from develop for all changes. Run verification: npm run lint && npm test && make scan-bad-imports

    This project uses @zebbra scoped packages from GitHub Packages. Authenticate with: npm login --scope=@zebbra --registry=https://npm.pkg.github.com (requires GitHub PAT with read:packages scope)
